## üìù Contribution Process

### Step 1: Fork & Clone
```bash
git clone https://github.com/YOUR-USERNAME/AIOps-Bootcamp.git
cd AIOps-Bootcamp
git remote add upstream https://github.com/pxkundu/AIOps-Bootcamp.git
```

### Step 2: Create Branch
```bash
git checkout -b fix/typo-week-03
# or
git checkout -b feature/new-exercise
```

### Step 3: Make Changes
- Follow our style guidelines (below)
- Test your changes
- Update documentation if needed

### Step 4: Commit
```bash
git add .
git commit -m "fix: correct typo in week-03 README"
```

Use conventional commits:
- `fix:` for bug fixes
- `feat:` for new features
- `docs:` for documentation
- `refactor:` for code refactoring

### Step 5: Push & Create PR
```bash
git push origin fix/typo-week-03
```

Then open a Pull Request on GitHub.

## üìã Style Guidelines

### Markdown
- Use ATX headers (`#`, `##`, `###`)
- One sentence per line (for easier diffs)
- Use fenced code blocks with language hints

### Code
- Python: Follow PEP 8
- Include docstrings
- Add comments for complex logic
- Provide example usage

### Exercises
- Clear objectives
- Step-by-step instructions
- Expected outcomes
- Common troubleshooting
